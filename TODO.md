เอกสารฉบับนี้รวบรวมรายการงานทางเทคนิคที่จำเป็นต้องได้รับการแก้ไขและรันเพิ่มเติมนบนสมุดบันทึกโค้ด (`plants.ipynb` หรือ `data.ipynb`) เพื่อให้ผลลัพธ์การฝึกสอนของโมเดลและการตั้งค่าพารามิเตอร์จริง สอดคล้องตรงกับเนื้อหาที่ได้ระบุแก้ไขไว้ในร่างบทความวิจัย ([RESEARCH_PAPER_DRAFT.md](RESEARCH_PAPER_DRAFT.md))

## 📂 1. การจัดการและคัดกรองชุดข้อมูล (Dataset Preparation)
- [ ] **คัดเลือกพันธุ์พืช 140 คลาส**: เขียนสคริปต์กรองพืชเหลือเพียง **140 ชนิดพันธุ์** โดยคัดเอาเฉพาะคลาสที่มีรูปภาพคุณภาพดีและมีภาพครบ 100 ภาพเป็นหลัก
- [ ] **ทำชุดข้อมูลที่มีความสมดุล (Balanced Dataset)**: จำกัดและคัดเลือกภาพให้เหลือ **100 ภาพต่อคลาสพอดี** สำหรับพืชทั้ง 140 ชนิด เพื่อจัดเตรียมชุดข้อมูลฝึกสอนรวมจำนวน **14,000 ภาพ**
- [ ] **การทำ Train/Validation Split**: แบ่งชุดข้อมูลในสัดส่วน ฝึกสอน (Train) 80% (11,200 ภาพ) และชุดตรวจสอบ (Validation) 20% (2,800 ภาพ)

## ⚙️ 2. การปรับโครงสร้างและการฝึกสอนโมเดล (Model Architecture & Training)
- [ ] **เปลี่ยนจำนวน Output Class**: แก้ไขโครงสร้างชั้น Dense Layer จำแนกผลตัวสุดท้าย (Classifier Node) จากเดิม 166 คลาส เป็น **140 คลาส** (พร้อมเปิดใช้งาน Softmax Activation)
- [ ] **เพิ่มระบบ Data Augmentation**: เพิ่มสคริปต์การทำ Augmentation ในโมเดล Keras ก่อนป้อนเข้าสู่การฝึกสอน:
  - สุ่มหมุนภาพ (Random Rotation) ในช่วง $\pm 15$ องศา
  - สุ่มซูมภาพ (Random Zoom) ในช่วง $\pm 10\%$
  - สุ่มปรับค่าความสว่าง (Random Brightness) ในช่วง $\pm 10\%$
  - พลิกรูปภาพตามแนวนอน (Horizontal Flip)
- [ ] **กำหนดขอบเขตในการ Fine-Tuning**: 
  - ตรึงค่าน้ำหนัก (Freeze) เลเยอร์โครงข่ายหลักทั้งหมดใน Epoch 1-50
  - ปลดล็อกเลเยอร์หลักบางส่วน (Unfreeze) ตั้งแต่ Epoch 51 เป็นต้นไป เฉพาะบริเวณ **Last Convolutional Block** ของโมเดลฐานแต่ละตัว
- [ ] **ปรับแต่ง Learning Rate**:
  - ขั้นตอนสกัดฟีเจอร์ (Feature Extraction): ตั้งค่า Learning Rate เริ่มต้นเป็น **$10^{-4}$**
  - ขั้นตอนการปรับแต่งเชิงลึก (Fine-Tuning): ปรับลดค่า Learning Rate ลงเหลือ **$10^{-5}$** เพื่อป้องกันไม่ให้น้ำหนักที่จดจำจาก ImageNet เสียหาย

## 📊 3. การรันการทดลองและการบันทึกผล (Experiments & Logging)
- [ ] **การฝึกสอนแบบจำลองใหม่ (Retraining Models)**: รันการฝึกสอนโมเดลใหม่ทั้งหมดสำหรับทั้ง 3 สถาปัตยกรรม:
  - [ ] MobileNetV2
  - [ ] EfficientNetB0
  - [ ] ResNet50V2
- [ ] **การวิเคราะห์ข้อผิดพลาด (Error Analysis)**: เขียนสคริปต์เพื่อคำนวณ Confusion Matrix หรือสกัดดูข้อมูล **Top-5 คู่พืชที่โมเดลจำแนกสับสนที่สุด** เพื่อใช้ในการเขียนรายงานผลทางพฤกษศาสตร์เชิงลึก
- [ ] **สร้างกราฟผลการทดลองใหม่ (Figure 1 Update)**: สร้างและบันทึกรูปกราฟความแม่นยำและค่าสูญเสียใหม่หลังเสร็จสิ้นการเทรน บันทึกไฟล์ทับรูปเดิมที่ `data/dataset/plant_model_comparison_merged_plot.png` โดยกำกับแกน Epochs และ Metrics ให้ชัดเจน
- [ ] **อัปเดตผลลัพธ์ในบทความ**: นำตัวเลขประสิทธิภาพความแม่นยำที่ดีที่สุดจริงจากผลการรันใหม่ กลับมาเติมแทนที่ตัวเลขเดิมในตารางที่ 1 ของบทความวิจัย
